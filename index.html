REGRA #1: Sempre Abrir da Raiz

# CORRETO - Sempre assim: cd ~/copywriting-ecosystem && claude # ERRADO - Nunca assim: cd ~/copywriting-ecosystem/concursos/hacker && claude

Por que? Hooks e instrucoes dependem do diretorio. Local errado = comportamento imprevisivel.

O que sao "Gates"?

Bloqueios automaticos de qualidade. Se os criterios nao forem cumpridos, o sistema NAO avanca:

Gate Valida Bloqueio
Researchsynthesis.md + 4 summaries VOC + Competitors + Mechanism + Avatar Nao avanca para Briefing
Briefing 10 fases HELIX + MUP/MUS aprovados Nao avanca para Producao
Production Secao GATE em todas copys Copy rejeitada automaticamente
Anti-Homo (v6.0) Genericidade Score >= 8/10 Copy generica = refazer

Nivel Jarvis - Gate Enforcement

v6.0: O sistema NAO deixa voce errar

O que e o Nivel Jarvis?

Sistema de enforcement automatico que garante qualidade sem intervencao manual:

  • Bloqueia producao sem research/briefing completos
  • Detecta e rejeita copy generica automaticamente
  • Calcula Genericidade Score (minimo 8/10)
  • Impede cliches do nicho em qualquer copy

Gate #5: Production Gates

Bloqueia Write/Edit em production/ se gates nao passaram:

# Mensagem de bloqueio: BLOQUEADO - Research incompleto Missing: voc/summary.md, competitors/ads-library-spy.md # Escape (use com cautela): "--force-production crie criativos para X"

Gate #6: Anti-Homogeneizacao

Verificacao Threshold Acao se Falhar
Cliches do nicho 0 permitidos Warning + substituicoes
Palavras banidas 0 permitidas Warning + alternativas
Nome proprietario Obrigatorio Warning se generico
Genericidade Score >= 8/10 "concorrente poderia usar"

Genericidade Score

Score Significado Acao
1-3 Qualquer concorrente usaria REFAZER
4-5 Precisa adaptacao REVISAR
6-7 Relativamente unica OK
8-10 Impossivel confundir APROVADO

Cliches Proibidos (Nicho Concursos)

Proibido Substituir por
"metodo infalivel" Nome proprietario do metodo
"passe em menos tempo" Prazo especifico + contexto
"decoreba nao funciona" Problema especifico
"estudar de forma inteligente" Tecnica nomeada

Palavras banidas (todos nichos):

revolucionario, inovador, incrivel, inacreditavel, transformacao, jornada, desbloquear, empoderar, segredo (sem especificacao), metodo (sem nome proprio)

Troubleshooting

Problemas comuns e solucoes rapidas

"Nao sei onde parei" +
/ecosystem-guide # ou "em que fase esta concursos/hacker?" "leia concursos/hacker/project_state.yaml"
"Apify/Playwright timeout" +

Sistema tenta automaticamente: Apify -> Playwright -> Firecrawl -> WebSearch

# Se todos falharem: "extraia comentarios de [URL] usando Playwright" "busque quotes sobre [topico] no Reddit usando Firecrawl"
"Copy generica / sem personalidade" +
  • Verificar se tem secao GATE na copy
  • Se NAO tem -> sistema nao leu arquivos obrigatorios
  • Solucao: "refaca lendo os arquivos obrigatorios"
  • Se elementos sao genericos: "releia e extraia exemplos ESPECIFICOS"
"Gate bloqueado" +
# MCP Tool (v5.5+): validate_gate gate_type="research" offer_path="concursos/hacker" # Output mostra EXATAMENTE o que falta # Exemplo: "MISSING: voc/trends-analysis.md"
"Gate bloqueou producao" (v6.0) +
# Solucao 1: Completar research "complete o research para concursos/hacker" /squad-research concursos/hacker # Solucao 2: Escape (use com cautela) "--force-production crie criativos para X"
"Genericidade Score baixo" (v6.0) +

Solucao:

  • Adicionar nome PROPRIETARIO para o metodo
  • Incluir numeros ESPECIFICOS (nao arredondados)
  • Usar linguagem da VOC (nao generica)
  • Adicionar prova social ESPECIFICA (nomes, datas)
"refaca com mais especificidade - adicione nome do metodo e numeros"
"Contexto poluido / confuso" +
# Compactar (mantendo contexto importante): /compact # Limpar completamente: /clear # Rebootar com contexto fresco: "leia concursos/hacker/project_state.yaml e me diga o status"
