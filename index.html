1 Fundamentos

O que e o Ecossistema?

Um sistema que substitui o operacional humano na producao de copy. O humano se torna orquestrador estrategico: toma decisoes criticas enquanto o sistema pesquisa, preenche briefings e produz copy.

Pipeline: PESQUISA → HELIX BRIEFING → PRODUCAO → VALIDACAO

Divisao de Trabalho

Humano (5% do tempo)Sistema (95% do tempo)
Define visao da ofertaExtrai 200+ quotes VOC de 5+ plataformas
Escolhe MUP e MUS (Fases 5-6)Minera Ads Library com Scale Score
Aprova gates entre fasesPreenche 10 fases HELIX automaticamente
Ajustes finais de tom/vozProduz VSL + LP + Criativos + Emails
Aprovacao final para publicarValida consistencia e roda stress-test

O que sao "Gates"?

Bloqueios de qualidade entre fases. Se os criterios nao forem cumpridos, o sistema NAO avanca. Cada gate valida automaticamente:

  • Research Gate: synthesis.md + 4 summaries + confidence ≥70%
  • Briefing Gate: 10 fases HELIX + MUP/MUS STAND ≥80%
  • Production Gate: Secao GATE em todas copys + consistencia
# Validar gate manualmente: python3 scripts/validate-gate.py RESEARCH concursos/hacker # PASSED = pode avancar | BLOCKED = lista o que falta

REGRA #1: Sempre Operar da Raiz

# CORRETO - Sempre assim: cd ~/copywriting-ecosystem && claude # ERRADO - Nunca assim: cd ~/copywriting-ecosystem/concursos/hacker && claude cd ~ && claude

Por que?

  • Claude Code carrega CLAUDE.md do diretorio atual
  • Hooks dependem de $(pwd) para descobrir contexto
  • Path errado = instrucoes incompletas + hooks quebrados

Como especificar oferta? Passa no comando:

/squad-research concursos/hacker "crie criativos para concursos/gabaritando-lei-seca" "continue o briefing do concursos/concursa-ai"

Hierarquia de Contexto

Claude carrega instrucoes nesta ordem (de mais importante para menos):

1. ~/.claude/CLAUDE.md ← GLOBAL (autoritativo) 2. ~/copywriting-ecosystem/CLAUDE.md ← PROJETO 3. {nicho}/biblioteca_nicho_*.md ← NICHO (VOC consolidada) 4. {oferta}/CLAUDE.md ← OFERTA (overview)

Regra: Nunca criar CLAUDE.md em subdiretorios (cria conflitos).

+ Plugin MCP: copywriting-mcp

NOVO em v5.5: Plugin MCP especializado para copywriting com database SQLite local e validacao automatica.

O que e o copywriting-mcp?

Um servidor MCP (Model Context Protocol) que adiciona 7 ferramentas especializadas para copywriting, com persistencia local em SQLite.

Diferenca Skills vs MCP Tools:

  • Skills: Orquestram workflows completos (pesquisa → briefing → producao)
  • MCP Tools: Operacoes atomicas rapidas (buscar VOC, validar gate, revisar copy)

4 Worktrees e Paralelizacao

O que sao Git Worktrees?

Worktrees permitem ter multiplas branches abertas simultaneamente em diretorios separados, sem precisar fazer checkout.

# Sem worktrees: precisa trocar branch git checkout feature-x ← Muda TODOS os arquivos do diretorio # Com worktrees: dois diretorios, duas branches ATIVAS ~/projeto/ ← branch main ~/projeto-feature-x/ ← branch feature-x (worktree) # Pode trabalhar em ambos ao MESMO TEMPO

Como claude-squad usa Worktrees

O /squad-research cria 4 worktrees automaticamente:

~/copywriting-ecosystem/ ← main (orquestrador) ~/copywriting-ecosystem-voc/ ← worktree (agente VOC) ~/copywriting-ecosystem-competitors/ ← worktree (agente Competitors) ~/copywriting-ecosystem-mechanism/ ← worktree (agente Mechanism) ~/copywriting-ecosystem-avatar/ ← worktree (agente Avatar)

Cada agente escreve em seu worktree → ao final, faz merge automatico no main.

Voce nao precisa gerenciar isso. O sistema cria e limpa automaticamente.

Aspecto/helix-parallel (Seq.)/squad-research (Par.)

Tenho deadline apertado? SIM → /squad-research (2-3x mais rapido) NAO → Continue... E minha primeira oferta no ecossistema? SIM → /helix-parallel (mais facil de aprender) NAO → Continue... Budget de tokens apertado? SIM → /helix-parallel (1x tokens) NAO → /squad-research (mais rapido)

5 Workflow Passo-a-Passo

ANTES DE TUDO

cd ~/copywriting-ecosystem && claude # SEMPRE comecar daqui. Nunca de subdiretorio.

FASE 0: Setup (5 min - Humano)

# Opcao 1: Slash command /ecosystem-guide # Opcao 2: Prompt natural "crie estrutura para oferta Hacker do Concurso no nicho concursos"

Verificar: Pasta criada com research/, briefings/, production/

Proximo: "existe biblioteca para o nicho concursos?" → Se sim, sistema carrega antes do research

FASE 1: Research (30-180 min - Sistema)

# Escolher um (da raiz!): /squad-research concursos/hacker ← Paralelo (rapido) /helix-parallel concursos/hacker ← Sequencial (padrao)

O que o sistema faz sozinho:

  • VOC: Extrai 200+ quotes de YouTube, TikTok, Instagram, Reddit
  • Competitors: Minera Ads Library, calcula Scale Score
  • Mechanism: Pesquisa mecanismos validados no nicho
  • Avatar: Perfil psicografico com DRE Framework e sinestesia

Quando intervir:

  • Menos de 100 quotes → "Continue extraindo VOC de [plataforma]"
  • Scale Score < 10 em todos → "Expanda busca para keywords [X]"
  • Apify timeout → Sistema tenta Playwright automaticamente

Como saber se terminou:

# MCP Tool (v5.5+ - automatico) validate_gate gate_type="research" offer_path="concursos/hacker" # ou Script Python (legado) python3 scripts/validate-gate.py RESEARCH concursos/hacker # PASSED → Avancar | BLOCKED → Lista o que falta

FASE 2: Briefing HELIX (60-90 min - Sistema + Humano)

/helix-system-agent # ou "preencha briefing HELIX para concursos/hacker"

10 Fases:

FaseConteudoAutomacao
1-4Identificacao, Mineracao, Avatar, Angulos100% auto
5MUP (Mecanismo Unico do Problema)DECISAO HUMANA
6MUS (Mecanismo Unico da Solucao)DECISAO HUMANA
7-8Big Offer, Stack, Pitch70-100% auto
9-10Criativos, Validacao STAND100% auto

Nas Fases 5 e 6:

# Sistema apresenta 3 opcoes de MUP/MUS # Voce responde: "Opcao 2" "Combine opcoes 1 e 3" "Crie novo baseado em [sua ideia]"

Validar MUP/MUS:

/copy-critic "valide o MUP - o avatar realmente acredita?" # Esperar: STAND (ok) | REVISE (ajustar) | ESCALATE (refazer)

FASE 3: Producao (30-60 min - Sistema)

/produce-offer concursos/hacker

Produz automaticamente: VSL + LP (14 blocos) + 7 Criativos + 5 Emails

OBRIGATORIO em cada copy - Secao GATE:

## GATE: Arquivos Lidos - research/synthesis.md → Sinestesia: "acordar 3AM sem fome" - briefings/phases/fase05_mup.md → MUP: "Cortisol Cronificado" - briefings/phases/fase06_mus.md → MUS: "Reset Circadiano" ## Consistencia - MUP usado: Cortisol Cronificado ← matches briefing ✓ - VOC refletido: 3 quotes usadas ✓

Se copy vier SEM secao GATE → REJEITAR automaticamente. Pedir refacao.

FASE 4: Validacao (15-30 min - Sistema + Humano)

# Multi-modelo review /review-all concursos/hacker # Stress-test adversarial /copy-critic

Validacoes: STAND ≥85% + MUP/MUS consistente + VOC refletido + Compliance

Decisao final do humano:

  • Li a copy completa?
  • MUP/MUS fazem sentido para o avatar?
  • Linguagem esta natural (nao parece IA)?
  • Pronto para publicar?

7 Troubleshooting

"Nao sei onde parei"

# Opcao 1: Perguntar /ecosystem-guide "em que fase esta concursos/hacker?" # Opcao 2: Ver estado direto "leia concursos/hacker/project_state.yaml" # Opcao 3: Rodar validacao python3 scripts/validate-gate.py RESEARCH concursos/hacker

"Apify falhou / Playwright timeout"

# Sistema tenta automaticamente: # Apify → Playwright → Firecrawl → WebSearch # Se todos falharem: "extraia comentarios de [URL especifica] usando Playwright" "busque quotes sobre [topico] no Reddit usando Firecrawl"

"Copy generica / sem personalidade"

  • Verificar se tem secao GATE na copy
  • Se NAO tem → sistema nao leu arquivos obrigatorios
  • Solucao: "refaca lendo os arquivos obrigatorios"
  • Se elementos sao genericos: "releia e extraia exemplos ESPECIFICOS"

"Gate bloqueado"

# Opcao 1: MCP Tool (RECOMENDADO - v5.5+) validate_gate gate_type="research" offer_path="concursos/hacker" # Opcao 2: Script Python (legado) python3 scripts/validate-gate.py RESEARCH concursos/hacker # Output mostra EXATAMENTE o que falta # Exemplo: "MISSING: voc/trends-analysis.md" # Acao: "complete o trends-analysis para hacker"

"Contexto poluido / confuso"

# Compactar (mantendo contexto importante): /compact # Limpar completamente: /clear # Rebootar com contexto fresco: "leia concursos/hacker/project_state.yaml e me diga o status"

"Claude ignorou instrucoes"

  • Verificar se abriu da raiz (regra #1)
  • Verificar se ~/.claude/CLAUDE.md existe e esta correto
  • /compact se contexto esta muito grande
  • Repetir instrucao de forma mais explicita
